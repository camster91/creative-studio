Security

How we protect your data and how to report vulnerabilities.

BYOK by default

The most security-sensitive piece of data — your Gemini API key — never touches our server. It's stored in your browser's localStorage, scoped to the photogen.ashbi.ca origin, and used directly from your browser when calling Google's API.

Encryption

  • In transit — TLS 1.2+ on all public traffic (Let's Encrypt via Caddy)
  • At rest — generated images are stored on the server's filesystem. Disk encryption depends on the host's infrastructure. Self-hosting users can encrypt the underlying volume.
  • Passwords — hashed with Argon2id (memory cost 64MB, 3 iterations, 1 parallelism)

Access control

  • Rate limiting on all public endpoints (60 req/min per IP for /api/*, 10 req/min for /auth/*)
  • Per-user image library is accessible only with a valid session token
  • Public share links are unguessable 32-char tokens, not enumerable

Self-hosting

The full source is at github.com/camster91/creative-studio. The production deploy is a single Docker container behind Caddy. You can run it on your own infrastructure with no code changes.

Vulnerability disclosure

Found a security issue? Email security@ashbi.ca with:

  • Description of the issue
  • Steps to reproduce
  • Impact assessment (your best guess is fine)

We respond within 48 hours and ship a fix within 7 days for critical issues. We credit reporters in the changelog unless you ask to remain anonymous.
